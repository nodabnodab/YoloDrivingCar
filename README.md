가장 가볍고 안정적이기 때문. 

어노테이션 툴은 직접 개발했음. 
피처는 차선 사진이고 라벨은 그 사진의 파일명으로 효율화. 
라벨은 곧 차선의 조향값. 

차량이 빠른 속도로 달리기 때문에 메모리를 미친 듯이 절약했다. 
1.필요없는 이미지 부분 자르기 
2.흑백으로 변환하기 
3.이미지 크기 전체를 줄이기.
4.다양한 환경에서 촬영하고 데이터 증강 하기. 

라벨링도 귀찮아져서 자동 라벨링해주는 기구까지 본인이 직접 만듬. (ai가 어느 정도 해준 거에 내가 조금씩만 수정해주면 됨)

🚗 Self-Driving Car: Jetson 기반 비전 AI 자율주행 및 임베디드 제어 시스템
📖 프로젝트 개요 (About the Project)
메인 보드가 장착된 RC 자동차에 NVIDIA Jetson 보드와 카메라를 연동하여 전동 모터를 실시간으로 제어하는 1인 풀스택 자율주행 프로젝트입니다. Vision 정보를 실시간으로 파악하고 그에 맞춰 앞바퀴 모터의 조향과 속도를 수정하는 임베디드 소프트웨어 및 AI 파이프라인 최적화 역량에 집중했습니다.
본 저장소는 완전히 다른 AI 모델을 활용한 두 가지 자율주행 서브 프로젝트를 통합하여 관리하고 있습니다.

객체 추적 (Object Tracking): YOLOv4 & TensorRT 기반 표지판 추종.
라인 추적 (Line Tracking): MobileNetV2 기반 도로 차선 추종 및 교차로 인식.

✨ 핵심 프로젝트 상세 (Key Projects)
🚩 Project 1. 객체 추적 자율주행 (YOLOv4 & TensorRT)
실시간 객체 탐지: 카메라 앞의 화살표 표지판 타겟을 YOLOv4 모델이 탐지하고 객체의 중심 위치에 따라 바퀴의 회전 각도를 즉각적으로 조향합니다.
TensorRT 최적화: Jetson의 제한된 환경에서 실시간성을 확보하기 위해 학습된 모델을 ONNX를 거쳐 TensorRT 엔진으로 변환, 최대 15 FPS의 추론 속도를 확보했습니다.
P-Control 제어: 객체의 중심 X 좌표와 화면 중앙 간의 오차를 계산하여 비례 제어(Proportional Control) 방식으로 차량의 조향값을 안정적으로 결정합니다.

🛣️ Project 2. 라인 추적 자율주행 (MobileNetV2 & Hybrid Control)
경량 AI 다중 출력 모델: 연산량을 줄이기 위해 MobileNetV2 기반의 경량 CNN 모델을 설계했으며 하나의 이미지에서 조향값(회귀)과 교차로 종류(분류)를 동시에 예측합니다.
직접 제작한 커스텀 데이터 라벨링 툴 🛠️: 기존 범용 툴의 비효율성을 극복하고자 OpenCV 기반의 어노테이션 툴을 직접 개발했습니다. 
마우스 클릭으로 라인 중앙점을 찍어 조향값을 추출하고 키보드로 교차로 종류를 라벨링하여 이 값들을 이미지 파일명에 직접 인코딩하는 방식

하이브리드 제어 (State Machine): AI의 순간적인 오판독을 방지하기 위해 상태 머신과 슬라이딩 윈도우 필터를 도입, 규칙 기반 제어와 AI 예측을 결합하여 교차로 구간에서 안정적인 주행을 구현했습니다.

🚀 실행 환경 및 가이드 (Getting Started)
이 프로젝트는 철저하게 NVIDIA Jetson 임베디드 환경에 맞추어 설계되었습니다.
단순한 PC 환경의 AI 예측 코드가 아니며 실시간 비전 데이터를 받아 Jetson 보드가 모터 드라이버(pop.Pilot 라이브러리 활용)로 앞바퀴 조향을 즉각 수정하는 제어 로직이 강하게 결합되어 있습니다.
실제 주행 테스트를 포함한 전체 코드를 구동하기 위해서는 카메라 및 모터를 제어할 수 있는 하드웨어 섀시와 Jetson 환경이 필수적으로 요구됩니다.

💡 트러블슈팅 및 성능 최적화 (Troubleshooting)
좌표계 불일치 해결: TensorRT 엔진의 출력 좌표가 Jetson 제어 시스템(원본 이미지)의 스케일과 일치하지 않는 문제를, 패딩 값과 축소 비율(Scale)을 역산하는 복원 수식으로 직접 구현하여 정밀한 좌표 기반 제어를 이뤄냈습니다.
멀티스레딩 기반 비동기 시스템 통합: 단일 스레드 환경에서 발생하는 블로킹 및 PyCUDA 컨텍스트 오류를 극복하기 위해 탐지 루프와 차량 제어 루프를 분리하고 PyCUDA 초기화를 백그라운드 연산 스레드 내부에서 실행하도록 아키텍처를 재설계했습니다.
